# SLIDE 8: LA LÓGICA INTELIGENTE - EFFICIENT FRONTIER (13:00-16:00)

```
═══════════════════════════════════════════════════════════
      EL CORAZÓN: CÓMO PORTALIOAIA RECOMIENDA
═══════════════════════════════════════════════════════════

PREGUNTA: ¿Cómo decide PortafolioIA qué inversiones recomendar?

RESPUESTA: Usa un algoritmo matemático avanzado:
"EFFICIENT FRONTIER" (Frontera Eficiente)

Este algoritmo fue desarrollado por Harry Markowitz en 1952
(ganó Premio Nobel por esto)

¿QUÉ HACE?

Imagina que tienes estos activos disponibles:

ACCIONES (QQQ, VOO, VTI)
  ✓ Alto retorno: +15% anual promedio
  ✗ Alto riesgo: suben/bajan mucho (-30% a +40%)

BONOS (BND, TLT)
  ✓ Bajo riesgo: muy estables
  ✗ Bajo retorno: +3% anual promedio

ORO (GLD)
  ✓ Seguro ante crisis económicas
  ✗ Retorno bajo

PREGUNTA: ¿Qué hago? ¿Todo en acciones (riesgo)? 
          ¿Todo en bonos (aburrido)? 
          ¿Algo en el medio?

RESPUESTA: MEZCLA INTELIGENTE

El algoritmo Efficient Frontier COMBINA:

┌──────────────────────────────────────────┐
│ Input                                    │
├──────────────────────────────────────────┤
│ 1. Tu perfil de riesgo (cuestionario)    │
│ 2. Retornos históricos de cada activo    │
│ 3. Volatilidad de cada activo            │
│ 4. Correlación entre activos             │
│    (¿Se mueven juntos o en direcciones   │
│     opuestas?)                           │
└──────────────────────────────────────────┘
         ↓ (Algoritmo corre matemática)
┌──────────────────────────────────────────┐
│ Output                                   │
├──────────────────────────────────────────┤
│ RECOMENDACIÓN FINAL:                     │
│ - 40% QQQ (Acciones Tech - crecimiento)  │
│ - 30% BND (Bonos - seguridad)            │
│ - 20% GLD (Oro - diversificación)        │
│ - 10% VNQ (Inmuebles - estabilidad)      │
│                                          │
│ Retorno esperado: 12.5% anual            │
│ Riesgo (volatilidad): 15%                │
│ Ratio Sharpe: 0.85 (bueno)               │
└──────────────────────────────────────────┘

¿POR QUÉ ESTO ES MEJOR QUE RANDOM?

❌ ANTES (Sin algoritmo):
   - Usuario escucha "Apple es buena inversión"
   - Mete todo en Apple
   - Apple baja → Usuario pierde todo
   - Usuario abandona, nunca invierte de nuevo

✅ CON ALGORITMO:
   - Usuario responde cuestionario
   - Algoritmo analiza 16+ activos
   - Diversifica: un poco en cada cosa
   - Si algo baja, otros suben
   - Retorno esperado + menor riesgo

EJEMPLO REAL:

Usuario CONSERVADOR (miedo al riesgo):
RECOMENDACIÓN:
- 50% Bonos (seguridad)
- 30% Acciones USA (crecimiento moderado)
- 20% Oro (seguro ante crisis)
Retorno: 6% anual, Riesgo: 8%

Usuario AGRESIVO (tolera riesgo, quiere crecer):
RECOMENDACIÓN:
- 50% Acciones Tech (QQQ)
- 30% Acciones Emergentes (Brasil, India)
- 15% Criptomonedas (BTC, ETH)
- 5% Oro (pequeña protección)
Retorno: 18% anual, Riesgo: 28%

═══════════════════════════════════════════════════════════

¿Y SI QUIERO EXPLICACIÓN?

Si está disponible Google Gemini (IA), genera:

"Te recomendamos esta cartera porque:

1. Tu edad (35 años) y horizonte temporal (10 años) 
   te permiten tolerar volatilidad

2. Tu ingreso mensual ($3,000) significa que puedes 
   rebalancear si es necesario

3. Respuestas al cuestionario indican perfil MODERADO:
   buscas crecimiento pero con protección

4. Efficient Frontier calculó que esta mezcla 
   maximiza retorno minimizando riesgo 
   (Ratio Sharpe de 0.85)

5. Historicamente, esta cartera ha ganado 12.5% anual 
   con variaciones de ±15%"

Todo en LENGUAJE SIMPLE, sin tecnicismo.

═══════════════════════════════════════════════════════════
```

**QUÉ DECIR:**
"Ahora voy a explicar el secreto: ¿cómo PortafolioIA 
decide qué invertir?

Usa un algoritmo llamado Efficient Frontier, que fue 
desarrollado por un Nobel de Economía hace 70 años. No 
es cosa nuestra, es CIENCIA.

El algoritmo pregunta: 'Tengo 16 activos diferentes - 
acciones, bonos, oro, etc. ¿Cuál es la MEJOR mezcla 
para maximizar ganancia y minimizar riesgo?'

Piensen en un jardín: si planto solo tomates, una plaga 
se come todo. Si planto tomates, papas, acelga, maíz, 
y un día hay plaga de tomates, al menos tengo otras 
cosechas. ESO es diversificación.

El algoritmo hace eso con dinero. Un usuario conservador 
recibe muchos bonos (seguro) y poco tech (riesgo). Un 
usuario agresivo recibe mucho tech y criptomonedas.

Y si la IA está disponible, genera una explicación 
personalizada: 'Te recomendamos esto porque tu edad es 
35, tu experiencia es principiante, y tus respuestas 
muestran que toleras este nivel de riesgo.'

Total: recomendaciones científicas, personalizadas, 
explicadas en palabras simples."

# SLIDE 11: SEGURIDAD Y AUTENTICACIÓN (19:00-21:00)
**Duración:** 2 minutos

```
═══════════════════════════════════════════════════════════
          SEGURIDAD: PROTEGIENDO A LOS USUARIOS
═══════════════════════════════════════════════════════════

🔐 AUTENTICACIÓN JWT (JSON Web Token)

FLUJO:
1. Usuario logea con email + password
2. Backend valida
3. Backend crea JWT: 
   {
     "user_id": 123,
     "email": "juan@example.com",
     "role": "user",
     "exp": "2026-05-20 10:30" → Expira en 24h
   }
4. JWT se CIFRA con una clave secreta del servidor
5. Retorna JWT al frontend
6. Frontend guarda en localStorage

EN CADA PETICIÓN:
Frontend añade: Authorization: Bearer [JWT]
Backend valida:
  ✓ JWT tiene formato correcto
  ✓ Fue cifrado con nuestra clave secreta (no falsificado)
  ✓ No expiró
  ✓ user_id existe en BD

SI FALLA:
→ Error 401 Unauthorized
→ Frontend redirige a login

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔑 CONTRASEÑAS (Hasheadas, no almacenadas)

FLUJO INCORRECTO ❌
Usuario: password = "MiPassword123"
BD: password = "MiPassword123"
PROBLEMA: Si alguien hackea BD, obtiene contraseña en TEXTO PLANO

FLUJO CORRECTO ✅
Usuario: password = "MiPassword123"
Backend: Aplica función hash (irreversible)
         "MiPassword123" → "$2b$12$KIXxPfx"
BD: password = "$2b$12$KIXxPfx" (HASHEADA)
RESULTADO: Si alguien hackea BD, obtiene hash inútil
           (imposible recuperar contraseña original)

VALIDACIÓN DE PASSWORD:
Usuario logea: "MiPassword123"
Backend: 
  1. Busca usuario en BD
  2. Obtiene hash: "$2b$12$KIXxPfx"
  3. Aplica hash a "MiPassword123" → "$2b$12$KIXxPfx"
  4. ¿Coinciden? SI → Login OK
              NO → Login FALLA

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

10. Si falla (pasó 1h): "Link expirado, intenta de nuevo"

✓ Token válido solo 1 hora
✓ Token no se reutiliza (invalida después)
✓ Link enviado solo a email registrado
```
