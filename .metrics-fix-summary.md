# Métrica Fix - Session Summary

## Problema Identificado
Los valores de métricas del portafolio (retorno esperado y nivel de riesgo) no cambiaban según las respuestas de la encuesta. Mostraban siempre "16.08%" y "NaN%" independientemente del perfil de riesgo del usuario.

## Root Cause (Causa Raíz)
Había **inconsistencia en la estructura de métricas** devueltas por diferentes funciones:
- `generate_portfolio_random()` devolvía: `{"expected_return": ..., "volatility": ...}` (usa "volatility")
- `generate_portfolio_static()` devolvía: `{"expected_return": ..., "risk": ...}` (usa "risk")
- `calculate_metrics()` devolvía: `{"expected_return": ..., "volatility": ...}` (usa "volatility")

Cuando el frontend intentaba acceder a `metrics.risk`, obtenía `undefined` que se convertía en `NaN%` al multiplicar por 100.

## Soluciones Implementadas

### 1. **Función normalize_metrics() (backend/app/services/optimizer_service.py)**
```python
def normalize_metrics(metrics: Dict[str, Any]) -> Dict[str, Any]:
    """Normaliza métricas a estructura estándar con campos validados"""
    normalized = {
        "expected_return": float(metrics.get("expected_return", 0.0)) or 0.0,
        "risk": float(metrics.get("risk", metrics.get("volatility", 0.0)) or 0.0),
        "sharpe_ratio": float(metrics.get("sharpe_ratio", 0.0)) or 0.0,
        "max_drawdown": float(metrics.get("max_drawdown", 0.0)) or 0.0
    }
    # Reemplazar NaN con 0
    for key in normalized:
        if str(normalized[key]) == 'nan' or normalized[key] is None:
            normalized[key] = 0.0
    return normalized
```

**Beneficios:**
- Convierte automáticamente "volatility" a "risk"
- Reemplaza NaN con 0.0
- Valida y estandariza la estructura de salida

### 2. **Actualización de generate_portfolio_random()**
Ahora applica `normalize_metrics()` antes de devolver:
```python
normalized_metrics = normalize_metrics(metrics)
result["metrics"] = normalized_metrics  # Asegura estructura consistente
```

### 3. **Mejora de portfolio.py (recalculación de métricas)**
```python
# Normalizar para consistencia (convierte volatility a risk)
normalized_metrics = normalize_metrics(real_metrics)
optimized_portfolio["metrics"] = normalized_metrics

# En caso de error, también normaliza metrics de fallback
if optimized_portfolio.get("metrics"):
    optimized_portfolio["metrics"] = normalize_metrics(optimized_portfolio.get("metrics", {}))
```

**Cambios:**
- Import de `normalize_metrics` desde optimizer_service
- Logging detallado para debug
- Manejo robusto de excepciones
